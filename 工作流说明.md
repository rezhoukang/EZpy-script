# JPG 文件整理工具 — 工作流说明

## 概述
本工具用于将选定根目录（最浅层路径）下的 JPG 文件，按文件名关系自动移动到其对应的子文件夹中。核心脚本：[jpg_organizer.py](jpg_organizer.py)。

## 前置环境
- Windows 系统
- Python 3.8+
- tkinter（Windows 下 Python 通常自带）
- 打包为 exe（可选）：`pyinstaller`

安装 PyInstaller：
```powershell
pip install pyinstaller
```

## 快速运行
- 以 GUI 方式运行：
```powershell
python "c:\Users\Desktop\新建文件夹 (3)\jpg_organizer.py"
```
- 在 Python 中直接调用（用于自动化或测试）：
```python
from jpg_organizer import scan_and_move
scan_and_move(r"C:\path\to\root_folder")
```

## 打包为单文件 exe
```powershell
pyinstaller --onefile --windowed --name "JPG文件整理工具" "c:\Users\Desktop\新建文件夹 (3)\jpg_organizer.py"
```
生成的可执行文件位于 `dist\JPG文件整理工具.exe`。

## 输入输出说明
- 选定根目录（最浅层路径）为扫描入口：
  - 根目录下的 JPG（直接位于根目录，不进入子目录）视为“b 文件组”。
  - 根目录下所有子目录（以及子目录的子目录）会被扫描，用来查找命名为“a接b(数字).JPG”形式的文件。
- 报告：每次执行结束都会在根目录生成 `时间戳_执行报告.txt`，包含每个 b 的处理结果（三类：成功移动、多个候选文件夹、无候选文件夹）。

## 名称解析规则（关键）
- 识别 JPG 文件扩展名不区分大小写（`.jpg`, `.jpeg`）。
- 任何文件名末尾连续出现的 `(数字)` 或 `（数字）` 都会被循环剥离（例如 `A (2)(1)` → `A`）。
- 子文件夹中文件的核心格式为 `a接b`（即文件基名以 `接` + `b` 结尾）。
- 匹配策略采用按 `b` 字符串长度从长到短优先，避免短字符串被误匹配。

## 移动逻辑
1. 收集根目录下所有 JPG，按去除末尾数字后得到的 `b` 分组。
2. 扫描子目录内所有 JPG，找到其基名以 `接{b}` 结尾的文件，记录该 `b` 对应的候选文件夹。
3. 对每个 `b`：
   - 若未找到候选文件夹：归入“未移动—未检测到可供移动的文件夹”并在报告中列出。
   - 若找到多个候选文件夹：归入“未移动—检测到超过一个可供移动的文件夹”，报告列出所有候选文件夹，等待人工处理。
   - 若恰好一个候选文件夹：将根目录下的该 `b` 所有文件移动到该候选文件夹中。
4. 如果目标文件夹已存在同名文件，移动时按 `(1)`、`(2)` 依次递增追加到文件名末尾（例如 `b.JPG` → `b(1).JPG`）。

## 报告格式示例
```
此次报告：
#1#  xxxx文件（x张）未移动，未检测到可供移动的文件夹
################################################
#2#  xxxx文件（x张）未移动，检测到超过一个的文件夹可供移动，分别为含有xxx文件的xxxx文件夹和含有xxx文件的xxxx文件夹和含有xxx文件的xxxx文件夹，请您自行移动
################################################
#3#  xxxx文件（x张）未移动，未检测到可供移动的文件夹
################################################
#4#  将xxxx文件（x张）移动到含有xxxx文件的文件夹xxxx中
################################################
over
```
报告文件保存在根目录，文件名为 `YYYYMMDD_HHMMSS_执行报告.txt`。


## 常见注意事项与扩展建议
- 若项目文件名包含其他杂项后缀（非 `(数字)`），需扩展解析规则。
- 可增加“预览/撤销”功能以防误动作。
- 支持 dry-run 模式（仅生成报告但不实际移动）可以先验证风险。
- 若目标环境不带 Python，可使用 PyInstaller 打包成单文件 exe 以便分发。
